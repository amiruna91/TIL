1.자바의 역사
  oak라는 언어에서 시작되어 초기 목적은 소형가전제품에 사용될 예정이였지만 기획이 무산되면서
추후 www ( world wide web) 및 인터넷이 대중화 되면서 운영체제데 독립적인 Oak가 인터넷에 적합
하게 개발되면서 자바(Java)로 변경되였다.

2.자바의 장점
   1)운영체제에 독립적이다. 
       windows, 매킨토시(Macintosh), 리눅스 등 운영체제에 상관없이 가동 된다.
   2)객체지향언어이다.
   3)비교적 배우기 쉽다.
   4)자동 메모리 관리(Garbage Collection)
      실행 순위가 낮기 때문에 약간에 딜레이가 생기지만 점점 발전 하고 있다.
   5)네트워크와 분산처리를 지원한다.
     풍부하고 다양한 네트워크 프로그래밍 라이브러리(Java API) 를 지원한다.
   6)멀티쓰레드르 지원한다
     멀티 프로세스와 멀티 쓰레드의 차이점은 :
          멀티 프로세스는 예로 들면 컴퓨터에서 음악을 들으면서 게임을 하는 즉 app 단위로 나뉘고
          멀티 쓰레드는 한 프로세스에서 동시에 작동 하는걸 말한다. (피파로 예를 들면 한명에 선수를
	 				동작하면 나머지 21명이 동시에 움직임을 말함)
   7)동적 로딩(Dynamic Loading)을 지원한다.
      자바는 여러 개의 클래스로 구성되어 있는데, 작동 할때 모든 클래스가 로딩되지 않고 그중 필요
    한 일부의 클래스만 작동 된다는 장점이 있다.

3.자바 가상 머신(JVM)
  자바가 운영체제에 독립적일수 있게 만들어 주는 역할을 한다.

4.자바 가상 머신(JVM)
  자바가 실행되는 가상 머신 이며 자바가 운영체제에서 독립적일수 있게 해준다.
  JVM은 번역기라 생각하면 되며 실행시 JVM을 거쳐 가기 때문에 다른 언어보다 느리다는 단점이 있다.
  하지만 바이트코드를 기계어로 변환해주는 JIT컴파일러가 향샹되여 그 격차가 많이 줄었다.

5.자바 개발 도구
  주로 JDK8 버전을 사용하며 JDK와 JRE로 나누워져 있다.
  JRE 안에 JVM, 라이브러리가 포함 되여있다.
  Java SE -> Desktop
  Java EE -> Server
  Java ME -> Small Device

6.자바 환경변수 설정
  윈도우 시스템에 환경변수로 (JAVA_HOME 경로) 를 만들어서 자바의 버전을 나타낸다.
  그리고 Path 에 %JAVA_HOME%\bin; 을 편집하여 실행파일을 실행시 경로를 따로 입력하지 않게 만들어 준다.

7.Java 기본 구조
  class 클래스이름 { <-여는 괄호
       public static void main(String[] args) <-- main 선언부{  
		수행문 ( 항상 마무리는 ;)
	}
} <-닫는 괄호

  1)System.out - 모니터 (표준 출력 장치)
    System.in  - 키보드 (표준 입력 장치)
    System.err - 모니터 (표준 에러 장치)

  2)method
    method 는 기능을 정의 한다.
    반환타입 메소드명(인자) {
	수행문;
    }
  
  3)main method
    main method는 JVM이 처음으로 실행하는 메서드이다

8.이클립스
  이클립스는 JAVA를 개발 할때 통합 개발 환경을 제공하는 프로그램이다.
  통합 개발 환경 : 오류 확인, 자동 완성, 디버깅, 단위 테스트, 실행

  1)이클립스를 설치 한후 몇가지 초기 설정을 해줘야된다
    encoding을 utf-8로 바꿔줘야된다.
  2)이클립스 화면은 여러 개의 창들로 이루어져 있는데, 이 하나의 창을 뷰(view)라고 하고 이 뷰들로 구성된
   하면 전체를 퍼스펙티브(perspective)라고 한다.
