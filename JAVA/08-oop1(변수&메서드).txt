1.사용자정의 타입
  기본형 외에 프로그래머가 서로 관련된 변수들을 묶어서 하나의 타입으로 새로 추가한것을 "사용자정의 타입"
 이라고한다. --- java에서는 class가 곧 사용자정의 타입이다.

2.지역 변수 와 클래스 변수와 인스턴스 변수
 지역 변수: method의 수행문에서 정의된 변수, 생성된 블록이 끝나면 즉시 사라진다.(stack 영역에 생성된다.)
 
 인스턴스 변수: - 객체 안에 생성된 변수이다. 또 다른 이름으로는 멤버 변수, property (heap 영역에 생성된다.)
                - 객체마다 고유한 값을 가지는 변수이다.
 
 클래스 변수: 클래스 변수 앞에는 "static" 이 붙는다!!! 이는 고정된 값이므로 같은 설계도로 생성된 객체들은
      	     같은 값을 바라보게 되며 공유한다.
       *특징: - 객체 생성과 상관없이 사용할 수 있다, 상수이다. 
              - 상수를 생성할때는 식별자는 모두 대문자이며 두단어 이상 붙을때는 _ 언더바를 사용한다.
	      예) static final double RATE_OF_INTEREST = 0.021;
	      - 클래스가 생성되는 시기는 new를 사용하여 객체를 만들때
		그리고 클래스 변수를 사용할때 클래스명. 만 사용해도 클래스는 로딩이된다.

3.메서드(method - function)
  - 특정한 작업을 수행할 목적으로 여러개의 수행문을 모아둔것
  - 메서드는 선언부 와 구현부로 이루어져 있다.
  예) 반환타입 메서드이름 (타입변수명, 타입 변수명,....) {      <---- 선언부
	    -메서드 호출시 수행될 코드-  			<---- 구현부
      }
  - 선언부는 반환타입, 메서드명, 매개변수 로 이루어져 있다.
	  * int <-반환타입 add <-메서드명 (int x, int y) <- 매개변수 { };

  1)매개변수(parameter)   - 메서드가 작업을 수행하는데 필요한 값을 제공받기 위한 변수.
  2)반환타입(return type) - 메서드의 수행결과로 획득되는 값의 자료형. 단, 반환값이 없으면 'void'를 적는다.
  3)return문 : 메서드의 반환타입이 'void'가 아닌경우, 구현부 안에 'return 반환값;'이 반드시 포함되야 된다.
