1.객체지향 언어
  특징 : 1)코드의 재사용성이 높다. 그러므로 생산효율이 좋다.
	 2)코드의 관리가 용이하다.(유지보수가 쉽다)
	 3)신뢰성이 높은 프로그래밍을 가능하게 한다.

2.객체와 클래스
 1)객체: 실제로 존하는 모든 사물 또는 개념;
	 프로그래밍에서는 클래스에 정의된 내용대로 메모리에 생성된것을 뜻한다.
 2)클래스: 객체를 생성하는데 필요한 설계도.
   클래스를 정의하는 이유는 설계도를 통해서 제품을 만드는 이유와 같다. 하나의 설계도만 잘 만들어 놓으면
  여러 제품을 만들기는 쉽기 때문이다.

3.객체의 구성요서 - 속성과 기능
 1)속성(특징): 같은 설계도로 만들어진 객체를 구분할 수 있게 해준다.
 2)기능: 같은 설계도로 만들어진 객체는 기능은 똑같다.
 3) 클래스(class)	 객체(Instance,Object,Bean)
    멤버변수(variable) ----> 속성(property)
    메서드(method)     ----> 기능(function)
 4)클래스로부터 객체를 만드는 과정을 클래스의 인스턴스화(instantiate)라고 한다.
 5)클래스에서 바로 정의하는 변수를 멤버변수(variable) 라고 한다.
